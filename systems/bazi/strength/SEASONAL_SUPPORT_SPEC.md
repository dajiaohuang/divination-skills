# Project seasonal-support path v0.1

Status: engineering baseline pending independent expert approval. It is never selected implicitly.

This bounded path provides a reproducible alternative to free-form claims that a day master is “strong” or “weak.” It does not claim to represent every Ziping school and must not be combined with pattern, climate-adjustment, transformation, blind-school, or useful-god rules.

## Feature contract

From the validated four-pillar facts:

1. The day-master element and its generating resource element are called supportive elements.
2. The main hidden stem of the month branch contributes two points when its element is supportive.
3. Each branch containing a hidden stem with the day-master element contributes one root point.
4. Each non-day visible stem whose element is supportive contributes one point.
5. Scores of 0–2 are labeled `drain_control_dominant`, 3–4 `mixed_support`, and 5–9 `support_dominant`.

The label describes only this feature vector. It is low-confidence interpretive metadata, not a personality, health, relationship, career, event, or useful-god conclusion.

## Isolation and review

The caller must explicitly pass lineage `project-seasonal-support-v0.1`. Reports record the selected lineage, all feature values, supporting fact IDs, and classifier rule ID. Until an external expert signs the rule set in M3, it is testable development output and not release-eligible guidance.
