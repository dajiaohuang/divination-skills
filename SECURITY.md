# Security and privacy

Do not report private user data in public issues. Birth dates, exact birth times, locations, relationship histories, and consultation notes may identify a person even after names are removed.

Report repository security issues privately to the repository owner. Until a private reporting channel is configured, do not attach exploit details or personal data to an issue; open a minimal issue requesting a secure contact channel.

Production deployments must define retention, export, deletion, encryption, access-control, logging, and incident-response policies before storing user data.
